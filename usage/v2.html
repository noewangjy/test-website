Hash values instead of integer IDs

Revised matcher API and phrase matcher

Example

Migrating from spaCy 1.x

Because weâ€™e made so many architectural changes to the library, weâ€™ve tried to keep breaking changes to a minimum. A lot of projects follow the philosophy that if youâ€™re going to break anything, you may as well break everything. We think migration is easier if thereâ€™s a logic to what has changed. Weâ€™ve therefore followed a policy of avoiding breaking changes to the Doc, Span and Token objects. If youâ€™ve trained models with input from v1.x, youâ€™ll need to retrain them with spaCy v2.0. All previous models will not be compatible with the new version.
